# Shortcut Bridge — Server

A tiny two-way relay between your web page and Apple Shortcuts.

## Endpoints

| Method | Path | What it does |
|--------|------|--------------|
| POST | `/messages/:session` | Web page sends a message |
| GET  | `/poll/:session`     | Web page polls for replies |
| POST | `/reply/:session`    | Your iPhone sends a reply back |

---

## Deploy free on Railway (recommended)

1. Go to https://railway.app and sign in with GitHub
2. Click **"New Project"** → **"Deploy from GitHub repo"**
3. Upload or push this folder to a GitHub repo first, then select it
4. Railway auto-detects Node and deploys it
5. Click your project → **Settings** → **Domains** → **Generate Domain**
6. Copy the domain (e.g. `shortcut-bridge-production.up.railway.app`)

---

## Wire up the web page

Paste this into the Webhook field on the Shortcut Bridge page:
```
https://YOUR-RAILWAY-DOMAIN.up.railway.app/messages/SESSION_ID
```
The session ID is auto-generated by the page — just paste the base domain and the page handles the rest.

Actually just paste:
```
https://YOUR-RAILWAY-DOMAIN.up.railway.app
```
And the page appends `/messages/{session}` automatically.

---

## Update Pipedream

Change your Pipedream workflow's ntfy notification to also include the session ID in the message body so you know which session to reply to.

Or skip Pipedream entirely — paste the Railway URL directly into the web page and it works standalone.

---

## Reply from your iPhone (Apple Shortcut)

Create a Shortcut with these actions:

1. **Ask for Input** — Prompt: "Reply to Shortcut Bridge"
2. **Get Contents of URL**
   - URL: `https://YOUR-RAILWAY-DOMAIN.up.railway.app/reply/SESSION_ID`
   - Method: POST
   - Headers: `Content-Type: application/json`
   - Body (JSON): `{"text": [Provided Input]}`
3. Done — the reply bubble appears on the web page within 3 seconds

Add this Shortcut to your Home Screen for one-tap replies.
